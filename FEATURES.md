# External Bridge — Feature Reference

A complete reference for every feature in the plugin.

---

## Bridge Management

### Add a Bridge
Configure a connection between an external folder on your disk and a destination folder inside your vault. Each bridge has its own settings for file types, subfolder scanning, custom metadata, and watching behaviour.

### Edit a Bridge
Modify any bridge after it's been created — change the label, external path, vault folder, file type filter, subfolder toggle, custom properties, or watcher toggle. The bridge is updated in-place; its sync history is preserved. If the watcher was running, it restarts automatically with the new config.

### Remove a Bridge
Deletes the bridge config and stops its watcher. Placeholder files already created in your vault are left untouched — they remain as normal notes.

### Health Indicator
Each bridge card shows a coloured dot next to the label indicating whether the external folder is currently reachable:
- **Green** — folder exists and is accessible
- **Red** — folder is missing or unreachable (e.g. an external drive that isn't mounted)

Unreachable bridges are also surfaced as a notice on Obsidian startup.

## Sync

### Manual Sync
Run **↻ Sync** on any bridge card to scan the external folder and create or update all placeholder files. A **↻ Sync All** button in the manager and a command palette entry (`Sync all bridges`) let you sync everything at once.

### Smart Sync (mtime-based)
Each placeholder stores the external file's last-modified timestamp (`external-mtime-ms`) in its frontmatter. On re-sync, the plugin compares this against the file's current mtime and skips the vault write entirely if nothing has changed. Large folders with hundreds of files re-sync in a fraction of the time after the first run.

### Live Sync Progress
While a sync is running, the Sync button updates in real time — `Syncing… 12/84` — so you know it's working on large folders rather than showing a static spinner.

### Auto-Sync on Startup
Optionally sync all reachable bridges automatically every time Obsidian opens. Unreachable bridges are skipped silently.

### Stale Placeholder Cleanup
After processing a bridge's external files, the plugin scans the vault folder for any placeholder that belongs to that bridge but whose source file no longer exists, and deletes it. Only files generated by this bridge are touched — your own notes in the same vault folder are never removed.

---

## File Watcher

Enable watching on a bridge to have placeholder files kept up to date in real time without manual syncs:

- **File added** → placeholder created immediately
- **File modified** → placeholder metadata updated, your notes preserved
- **File deleted** → placeholder removed from the vault

### Watcher Status Indicator
Each bridge card shows a second dot for watcher state:
- **Pulsing green** — watcher is active
- **Grey** — watcher is off

Watchers whose `watchEnabled` flag is set restart automatically when Obsidian reopens.

### Debounce
A configurable delay (default 2000 ms) prevents redundant updates when a file is being written incrementally. Configurable globally in settings.

### Polling Fallback
If `fs.watch` is unavailable (e.g. a network-mounted drive), the watcher falls back to polling the folder every 5 seconds.

## Placeholder Structure

Every placeholder is a standard Obsidian markdown note with three sections:

### 1. Frontmatter
Auto-generated YAML at the top of the file. Re-written on every sync.

```yaml
external-path: "/Users/you/Documents/Notes/session.pdf"
external-file: "session.pdf"
external-type: "pdf"
external-size: "4.2 MB"
external-modified: "2024-03-01"
external-mtime-ms: 1709251200000
bridge-id: "a3f9bc12"
bridge-label: "Handwritten Notes"
tags:
  - pdf
  - external-bridge
```

Custom properties defined on the bridge are injected here as additional frontmatter fields.

### 2. Meta Body
Auto-generated content block wrapped in HTML comment sentinels (`<!-- bridge:meta:start -->` / `<!-- bridge:meta:end -->`). Re-written on every sync. Contains:
- A heading with the filename
- An info table (type, size, modified date, source path)
- A link to open the file in the system viewer
- A type-specific preview block (see [File Type Previews](#file-type-previews))

### 3. User Content Zone
A clearly marked area between `<!-- bridge:user-content:start -->` and `<!-- bridge:user-content:end -->`. **Never overwritten on re-sync.** Add your own notes, wikilinks, tags, and properties here freely.

## File Type Previews

Each placeholder includes a preview block appropriate to its file type:

| Type | Preview |
|---|---|
| **PDF** | Callout box with a link to open in the system PDF viewer |
| **Images** (png, jpg, jpeg, gif, webp) | Inline `![]()` embed — renders the image directly in the note |
| **Audio / Video** (mp3, mp4, mov, m4a) | Callout box with a link to open in the system media player |
| **Office / Text** (docx, xlsx, pptx, txt, md) | Generic callout with a link to open in the default app |

## Custom Properties

Each bridge can inject additional YAML frontmatter fields into every placeholder it generates. Define them in the bridge config as `key: value` pairs, one per line:

```
subject: music
status: unreviewed
notebook: LittleBlackie
```

This makes it easy to query placeholders from a specific bridge using Obsidian's search, Dataview, or Bases — for example, filtering by `subject = music` or `status = unreviewed`.

---

## Open Folder Shortcuts

Each bridge card has two buttons to jump to the relevant folder without leaving Obsidian:

- **↗ External** — opens the external folder in Finder (macOS) or Explorer (Windows)
- **↗ Vault** — opens the vault placeholder folder in Finder / Explorer

---

## Sync Log

Every sync that creates, updates, or removes at least one placeholder is recorded in a per-bridge log accessible from the **📋 Log** button on the bridge card.

Each entry shows:
- Timestamp of the sync
- Badge counts: `+created`, `~updated`, `−removed`, `⊘skipped`
- Expandable detail list of which filenames were affected

### Log Limit
Configure a maximum number of entries to retain per bridge (default: unlimited). When the limit is set, older entries are trimmed automatically. The limit can be adjusted at any time from Settings.

---

## Settings

| Setting | Default | Description |
|---|---|---|
| **Default tags** | _(empty)_ | Tags added to every placeholder across all bridges, in addition to the file-type tag and `external-bridge` |
| **File watcher debounce (ms)** | `2000` | How long to wait after a file-change event before processing it |
| **Sync log limit** | `0` (unlimited) | Maximum sync log entries to keep per bridge |
| **Auto-sync on startup** | Off | Sync all reachable bridges when Obsidian opens |
| **Open Bridge Manager on startup** | Off | Show the Bridge Manager modal automatically on launch |

## Supported File Types

| Category | Extensions |
|---|---|
| Documents | `pdf` |
| Images | `png` `jpg` `jpeg` `gif` `webp` |
| Audio | `mp3` `m4a` |
| Video | `mp4` `mov` |
| Office | `docx` `xlsx` `pptx` |
| Text | `txt` `md` |

Each bridge can target any combination of these. Leaving all unchecked includes all types.
